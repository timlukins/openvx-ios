InVX is our freely available SDK for developing OpenVX applications. Hardware vendors will be providing OpenVX drivers for an increasing number of platforms but these are not available today so there is a problem if you wish to start developing high performance, portable vision applications right now.

We have developed InVX to solve this problem. InVX is a software implementation of the API that uses platform specific hardware acceleration to get performance. The implementation is built on top of Khronos Group's sample implementation of the API but with platform specific acceleration and other optimisations provided by us.

InVX is the first implementation of hardware accelerated OpenVX for mass market smartphones and IoT platforms. We use InVX in-house ourselves to build great solutions for customers who out-source their computer vision development to us.

Compatibility & Acceleration Status

InVX has not been conformance tested against the OpenVX specification so we do not claim 100% compatible with OpenVX today but we do carry out extensive testing of our implementation. We plan to address this in the future. We expect InVX to eventually be superceded by hardware vendor OpenVX drivers but this make take many years so if users of InVX have a requirement for full OpenVX compliance they should contact us. Each release of InVX includes a fully built version of the OpenVX 1.0 sample implementation (often referenced as libopenvx-c_model) which means it provides complete coverage of the API.
