Portable, Power-efficient Vision Processing.

Low cost cameras, powerful mobile computers and breakthroughs in vision algorithms are enabling new and enhanced products that understand the world around them. The challenge to fully realising this potential is maximising performance over a diverse range of computing platforms. Reasonable performance can usually be attained by optimising for a particular platform but at the expense of significant effort and loss of portability.

OpenVX has been developed to address these challenges. It is an open, royalty-free standard for cross platform acceleration of computer vision applications. The standard is overseen by the Khronos Group and has been developed by a large consortium of industry partners. The main site - with a more detailed overview - can be found at https://www.khronos.org/openvx/ and there is also an accompanying overview slide-set [PDF] as well.

In summary, here are some of the key features of OpenVX:

  • Designed for low-power, real-time applications.
  • Specifically targeted at mobile, embedded computing platforms.
  • Supports portable application code with a standardised API and implementation conformance tests.
  • Hides low level implementation details in an increasingly heterogeneous platform landscape.
  • Algorithms can be expressed as graphs allowing significant optimisation when mapping onto hardware.

Here and now.

OpenVX version 1.0 has only recently (as of the end of October 2014) been ratified. The complete specification is now available. Following this, OpenVX drivers will now be provided by hardware manufacturers for an increasing number of platforms. Some of the most exciting are:

  • Nvidia's implementation of OpenVX built on-top of CUDA (powered by the upcoming Tegra K1 processor). Here is a slide-set with a great example of tracking code written in OpenVX [PDF].
  • Google's project Tango, powered by Movidius' Myriad 1 vision processor.
  • Videantis' v-MP4000HDX Chip technology.

However, if you are interested in exploring and benefiting from OpenVX today your options are limited until wider adoption. Especially if you want to target existing mobile platforms such as Android and iOS, or IoT linux devices (such as the Raspberry Pi).

Machines with Vision is raising awareness of OpenVX and supporting developers by providing software implementations of the API that take advantage of hardware acceleration on these platforms. Please check out our InVX SDK if you want to deploy OpenVX technology, starting today, for mobile and IoT.

We are excited about OpenVX since we think it addresses a fundamental need for a unified, standardised framework for deploying embedded computer vision algorithms. This makes it applicable not just for mobile and embedded devices - but ultimately for vehicles, robotics, drones, and anything else which has sensors and a requirement to process visual information quickly and efficiently.
